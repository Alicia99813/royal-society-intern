Simple Site - Extensions

Adding further functionality

This project will take a small set of JSON files, process them and produce a formatted set of html webpages. So the next question is, what other processing functions can be added to the system to allow ones GitHub project to host additional tools or even turn a project into a simple virtual research environment.

New Opportunities

What jobs could required consistent processing and presentation of simple files of text or numbers and then present formatted results in images viewers, graphs, maps, flow charts, etc?

There are lots of great pieces of software out there, so this project is intended to make use of open source libraries rather than creating new software.

The main question here is, how can researchers, who do not have their own web server, or sufficient coding skills gain access to their own data processing and presentation system?

Beyond the standard variables that have already been explained the sub-page JSON file includes two additional variables that can be used to indicate the inclusion of a page extension.

Simply, the class variable indicates what type of extension should be included and the file variable indicates the name of the file that includes the required extra data. The two examples provided here are:

  • The timeline class, which can be used to include a formatted Gantt Chart, using the Mermaid, to display a simple project timeline.
  • The mirador class, which can be used to include an IIIF images viewer called Mirador, to organise and present high resolution images, which are already hosted elsewhere on the web.

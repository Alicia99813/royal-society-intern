Simple Site - The pages.json file

  1. Json

As noted on the main Json page you will need to ensure that your page is valid json, please ensure all double quotes in your added text have a / in front, like this /", also please ensure that you text is all on one line, with no newlines.

The pages.json file is used to define the actual content for your top level or main pages. The file is made up of one or more page templates, an example of which is shown below. The meaning of these fields are relatively straightforward, as described below, for an example of real code please see this example pages.json file.

Each of the values within the template can include simple text information or more complex html, however you must ensure that you follow guidelines included of the main Json page.

In addition to standard html coding some additional options have been added to make life easier. These do not need to be used if not needed.

  • Adding footnotes: Locate footnotes references in your text using [@@], then add all of the footnote content in a comma separated list within the footnotes field in the page template, see the example below. These will all be automatically processed so that all of the footnotes added to the content or content right are numbered correctly and clickable.
  • Adding links: Links can be added to any word or phrase simply by enclosing them in square brackets, separated with a pipe character: [Display text|https://full.url.com]
